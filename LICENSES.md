## License obligations we comply with

### Apache 2.0 dependencies
- Preserve the LICENSE file in distributions
- Preserve the NOTICE file when present
- Document modifications when forking

### MIT / BSD / ISC dependencies
- Preserve the copyright notice and license text in distributions

We propagate these via:
- Backend: `pip-licenses` collects all dep licenses into the deployment image
- Frontend: `vite-plugin-license` bundles license texts with the JS bundle (Phase 1 to add)

## Banned licenses (CI lint rule, Phase 1)

The following will fail CI when a new dependency is added:

- **Elastic License v2 (ELv2)** — restricts hosted-service distribution
- **Server Side Public License (SSPL)** — MongoDB-style; viral on derived services
- **AGPL v3** — copyleft over network use
- **GPL v2 / v3** (linking) — copyleft
- **BUSL / Business Source License** — time-bombed restrictions
- **Creative Commons Non-Commercial (CC-BY-NC, etc.)** — bans commercial use
- **Source-available, no-commercial-use** custom licenses

## How to audit when adding a dep

```bash
# Backend
pip install <package>
pip-licenses --packages <package>

# Frontend
npm view <package> license
```

If the result is anything other than MIT / Apache 2.0 / BSD-2 / BSD-3 / ISC / Unlicense / 0BSD / CC0:
- Stop
- Read the actual license text
- Decide: replace with permissive alternative, OR document a compelling reason to accept
